About

The Classificator is a high level API, wrapping around scikit-learn’s classification suite, whose primary function is to normalize and automate the process by which common machine learning classification models are trained and validated. This is specifically in the context of textual feature spaces, which require NLP techniques for preprocessing and vectorization. There is an additional focus on group cross validation, where labeled groups exist as data features, and variance is known to be smaller within groups.

Philosophical Motivation

Data democratization has been more or less achieved within large organizations since the advent of software like Hive, allowing anyone who can write SQL queries to derive insights from large distributed data sets without having to write MapReduce jobs. The same can be said for the democratization of hardware with the growing popularity of on-demand cloud services, offering self service distributed compute clusters using software like Amazon EMR and Qubole.

This theme of democratization is attractive, as lowering the barrier of entry into any field necessarily increases access for users within an organization, or as individuals, and removes bottlenecks that cause many projects to never leave the white board. In the spirit of rapid prototyping, a tool that democratizes text classification and removes the redundancy of writing the same code time and time again in a similar problem space, has the ability to vastly improve the number of projects considered and the speed at which concepts can be proven.

Technical Features

Load a dataset
- User points to a dataset locally or in Amazon S3
- User defines simple data specs
- User defines feature columns, label column and group columns

Load a configuration
- User specifies preprocessing options, model choices, and hyperparameter grids per model choice
- User specifies meta parameters

Train and validate various model choices
- Uses common cross validation and grid search techniques, and chooses the best model based on a user supplied scoring option
- Return validation statistics and save the model
- User defines output location for validation report and pickled model object

Comparison to Similar Services

There are many services that offer “machine learning as a service” or “data analytics as a service”.

The Classificator has some advantages:
- Tailored to text classification problem space
- Data is stored in-house for security
- Free
- Transparent
- This is not a black box service. The code is open source and is entirely built on top of scikit-learn with clear references to documentation and explanations of algorithms and hyperparameters
- User supplies hyperparameter grids for full control over testing and optimization
- Focus is on education and learning about different algorithms and their hyperparameters rather than on writing code. This doesn't replace a data scientist, it empowers them to be more productive and concentrate on algorithms and the feature space rather than on writing code

And some disadvantages:
- Rough UI (not a great UX but gets the job done)
- Less exhaustive
- Limited to scikit-learn (no tensorflow or focus on deep learning)
- Probably fewer high level options
- Less turn-key
- Will require some thought from the user, however sane defaults are used for a decent outcome for many text classification problems
- Some services offer a distributed service for speed. The Classificator could be deployed on multiple compute partitions in practice but there is no built in option for this
